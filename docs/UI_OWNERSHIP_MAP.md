# GATE UI Ownership Map

Status: Phase Zero baseline
Scope: Documentation only. No runtime behavior changes.

## Purpose

This document maps every major UI surface to its current owner, duplicate / overlapping owners, intended canonical owner, and recommended disposition.

GATE must move toward one owner per page, modal, workflow, and component. Current instability is caused by multiple active files controlling the same surfaces.

## Ownership rule

Each surface must ultimately be classified as one of:

- Canonical owner
- Compatibility layer
- Legacy source pending removal
- Inactive legacy
- Unknown / requires test

No surface should have more than one canonical owner.

## Global app shell and navigation

Current active owners:

- `public/index.html`
  - Defines base page markup, `showPage`, `buildNav`, role toggle behavior, and base nav buttons.
- `public/js/prc-dash-final-audit.js`
  - Adds command shell classes, inserts mobile menu trigger, patches `buildNav`, adds Squadron Board page option.
- `public/js/gate-access-control-controller.js`
  - Patches navigation, enforces role page permissions, guards pages/forms/actions.
- `public/js/gate-mobile-nav-routing-fix.js`
  - Intercepts mobile nav taps and routes by `data-page`.
- `public/js/gate-mobile-shell-redesign.js`
  - Moves system controls into mobile menu and changes shell layout on mobile.
- `public/js/gate-mobile-app-shell-finalizer.js`
  - Forces mobile page isolation, patches `showPage`, manages mobile drawer state.
- `public/js/gate-desktop-nav-restore.js`
  - Restores desktop navigation when mobile shell affects desktop.

Current risk:

- Multiple files patch navigation and page state.
- Mobile shell can affect desktop unless guarded.
- Page visibility can drift because different controllers use different active/hidden assumptions.

Target canonical owner:

- Future `AppShellRouter` plus `ResponsiveShell` plus `Permissions`.

Disposition:

- Consolidate first in Phase 1A.
- Retire navigation patching from `prc-dash-final-audit.js` after canonical router exists.
- Retire `gate-mobile-nav-routing-fix.js`, `gate-mobile-app-shell-finalizer.js`, and `gate-desktop-nav-restore.js` once responsive shell is canonical.

## Status Board metrics

Current active owners:

- `public/index.html`
  - Still writes `metric-arrived` and `metric-airport` from base render loop.
- `public/js/prc-dash-board-header.js`
  - Now compatibility-only; should remove old v3 duplicate metric grid if it appears.
- `public/js/gate-premium-metrics-controller.js`
  - Current canonical visible owner of four-card metric layout.
- `public/css/gate-premium-metrics.css`
  - Current visual style owner for premium metric cards.

Current risk:

- Legacy metric IDs are still written by base `renderAll()` and `updateAirportMetric()`.
- Compatibility sinks are required until base render loop stops writing old IDs.

Target canonical owner:

- Future `StatusMetrics` component inside `StatusBoardPage`.

Disposition:

- Keep `gate-premium-metrics-controller.js` temporarily as canonical visible owner.
- Remove old metric source writes from `index.html` during Status Board refactor.
- Delete compatibility sinks only after `renderAll()` no longer references `metric-arrived` or `metric-airport`.

  - Instructor right-click actions, record edit preservation, open/close record support.

  - Tablet modal scroll accessibility.

Disposition:

- Refactor after navigation and Status Board.
- Preserve backend dorm timing protections.
- Collapse all modal behavior into one modal controller.

Current risk:

- Mobile presentation, routing, menu, page visibility, and form layout are spread across many runtime files.
- Desktop regression risk is high.

Target canonical owner:

- Future `ResponsiveShell` with CSS-first layout and one drawer controller.

Disposition:

- Refactor first after Phase Zero.
- Do not add more mobile patch files unless production-stabilization requires it.
